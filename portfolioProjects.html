Player vs Skeleton

(Click the picture to see the code)

The thought behind this came as I was learning how objects work and how I can get two classes to interact with each others properties. By instantiating an object from each class I was able to simulate a battle between a player and the skeleton, creating new values each round to make the game less predictable whilst challenging the player to win in the least amount of turns possible.
